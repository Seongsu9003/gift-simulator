# 증여 시뮬레이터 - 서비스 고도화 플랜

> **기반 아이디어**: 20년차 기획 담당자의 4단계 스토리텔링 기획안
> **작성일**: 2026년 3월
> **목적**: "계산기"에서 "가족의 미래를 설계하는 도구"로의 전환

---

## 스토리텔링 핵심 구조

사용자 감정 흐름: **감성 → 공포(현실 직시) → 안도(해결책) → 행동(CTA)**

| 단계 | 감정 | 핵심 메시지 |
|------|------|-------------|
| 1단계 | 감성·공감 | "내 아이의 미래 목표가 뭐지?" |
| 2단계 | 공포·위기감 | "생각보다 훨씬 많이 필요하네" |
| 3단계 | 안도·희망 | "지금 시작하면 충분히 가능하다" |
| 4단계 | 행동 | "이 플랜으로 바로 시작하자" |

---

## Phase 1 — 즉시 적용 (1~2주)

> 난이도: 낮음 | 효과: 높음 | 추가 데이터 불필요

### 작업 1-1. 목표 선택 화면 신규 추가

**위치**: 로그인 → 자녀 선택 → **[목표 선택 화면 NEW]** → Step1 자녀정보

**화면 구성**

- 상단: "OO이가 {N}년 후 독립할 때, 가장 선물하고 싶은 것은?"
  - 자녀 이름 + 생년월일로 "몇 년 후"를 자동 계산
  - 예) 현재 5살 → "15년 후", 신생아 → "20년 후"
- 중앙: 3개 아이콘 카드 버튼
  - 대학 등록금 및 유학 자금
  - 첫 전셋집 마련 보증금
  - 사회초년생 시드머니
- 하단: "아직 모르겠어요 → 종합 플랜으로 시작" 탈출구 버튼 (이탈률 방지)

**구현 포인트**

```
// 신규 파일: client/src/components/GoalSelection.js
// 선택한 goal 값을 App.js의 state로 올려보내 전체 위자드에서 참조
const [selectedGoal, setSelectedGoal] = useState(null);
// 'tuition' | 'jeonse' | 'seedmoney' | 'general'
```

**UX 규칙**
- 이모지 미사용 (v1.2.0 원칙 준수)
- 선택 상태: 코랄 배경 (#FFF5F3) + 코랄 테두리 (#FF8C69)
- 아이콘은 SVG 또는 텍스트 기반으로 처리

### 작업 1-2. 선택한 목표를 위자드 전체에 반영

**변경 파일**: `StepWizard.js`, `TaxCalculator.js`, `Simulator.js`

- Step 2 상단 배너: "목표: 대학 등록금 마련을 위한 씨드머니 계획"
- Step 3 상단: "대학 등록금 목표 기준, {자녀 나이}세까지 예상 필요액: X억 원"

**데이터 플로우 변경**

```

[변경 후]
App.js → childInfo + selectedGoal → StepWizard.js → 하위 컴포넌트 전달
```

## Phase 2 — 단기 적용 (2~4주)

> 난이도: 중간 | 효과: 설득력 극대화

### 작업 2-1. 인플레이션 목표 금액 표시

**위치**: Step 2 씨드머니 계획, Step 3 예상 증여세 계산 결과 영역

**계산 로직**

```javascript
// 연 3% 고정 복리로 인플레이션 적용 (출처: 통계청 소비자물가지수 기반)
const INFLATION_RATE = 0.03;

function calcInflatedAmount(baseAmount, yearsUntilGoal) {
  return baseAmount * Math.pow(1 + INFLATION_RATE, yearsUntilGoal);
}

// 목표별 현재 가치 기준값 (2026년 기준, 주기적 업데이트 필요)
const GOAL_BASE_AMOUNTS = {
  tuition: 4000,      // 4년 대학 등록금 약 4,000만 원 (사립 기준)
  jeonse: 30000,      // 서울 평균 전세 보증금 약 3억 원
  seedmoney: 5000,    // 사회초년생 시드머니 약 5,000만 원
};
```

**UI 표시**
- "현재 가치 기준: 약 X만 원"
- "→ {목표연도} 인플레이션 반영 시: 약 Y억 원"
- 하단에 작은 글씨로 "연 3% 물가상승률 기준 추정치 | 출처: 통계청"

**주의사항**
- 예상 수치는 항상 "참고용 추정치이며 실제와 다를 수 있습니다" 면책 문구 포함
- 데이터 기준년도를 명시하고 연 1회 업데이트 체계 마련

### 작업 2-2. 예금 vs ETF 듀얼 비교 그래프

**위치**: Step 4 투자시뮬레이션 (기존 Simulator.js 확장)

**변경 내용**

현재 단일 수익률(10.23%) 그래프 → 두 개의 라인을 비교하는 듀얼 그래프로 전환

```javascript
// 기존: 단일 라인
const ETF_RATE = 0.1023;

// 변경: 두 라인 비교
const DEPOSIT_RATE = 0.035;   // 예금 연 3.5% 고정 (한국은행 기준금리 기반)
const ETF_RATE = 0.1023;      // S&P 500 역사적 평균 (변경 금지)

// Recharts에 두 개의 <Line> 추가
<Line dataKey="deposit" stroke="#9CA3AF" strokeDasharray="5 5" name="정기예금" />
<Line dataKey="etf"     stroke="#FF8C69" strokeWidth={2}       name="미국 ETF" />
```

**스토리텔링 문구 (그래프 하단)**

> "매월 20만 원씩 20년간 투자했다면 (2005~2025년 기준)"
> - 정기예금: 원금 4,860만 원 → 약 7,900만 원 (+62%)
> - 미국 S&P 500: 원금 4,860만 원 → 약 2.9억 원 (+502%)
>
> ※ 과거 수익률이 미래를 보장하지 않습니다. 참고용입니다.

**애니메이션**
- 두 그래프가 왼쪽에서 오른쪽으로 순차 그려지는 효과
- Recharts `animationDuration`과 `animationBegin` 속성으로 A 그래프가 먼저, B 그래프가 약간 늦게 시작

### 작업 2-3. 인플레이션 리얼리티 체크 화면 (선택 구현)

**위치**: 목표 선택(GoalSelection) 직후, Step 1 전

**구성**
- 배경 또는 카드: 짜장면 가격 변화 인포그래픽 (텍스트 기반)
  - 1970년: 100원 → 2000년: 2,500원 → 2025년: 8,500원
- 선택한 목표에 따른 핵심 데이터 1~2줄 제시
  - 전셋집 선택 시: "2015년 서울 평균 전세 5억 → 2025년 약 9억 (+80%)"
  - 대학 등록금 선택 시: "2015년 월 사교육비 23만 원 → 2025년 44만 원 (+91%)"
- 하단 결론 문구: "OO이가 {목표연도}에 필요한 금액은 약 X억 원으로 예상됩니다."
- CTA 버튼: "지금 준비하면 충분합니다 →"

**구현 우선순위**: Phase 2 중 가장 마지막에 구현 (임팩트는 크지만 콘텐츠 준비 시간 필요)

## Phase 3 — V2 적용 (3~6개월)

> 금융 상품 연동 및 전환율 고도화

### 작업 3-1. 맞춤형 액션 플랜 Bottom Sheet

**트리거**: Step 4 투자시뮬레이션 결과 확인 후 하단에서 슬라이드 업

**내용 구성**
- 자동 계산된 월 최적 납입액: "OO이의 목표 달성을 위해 월 {N}만 원이 필요합니다"
- 기준: 증여세 면제 한도 최대 활용 + 목표 금액 역산
- 유기정기금 절세 혜택 설명 (세무사 확인 필요 면책 문구 반드시 포함)

**CTA 버튼 (MVP 임시)**
- "홈택스 증여 신고 바로가기" (외부 링크)
- "이 플랜 PDF로 저장하기" (V2 PDF 기능과 연동)

**CTA 버튼 (V2 파트너 연동 시)**
- 금융사 API 연동 후 "지금 바로 시작하기" → 상품 가입 페이지로 이동

---

### 작업 3-2. 유기정기금 절세 계산기

**위치**: Step 3 예상 증여세 계산 탭에 섹션 추가

**기능**
- 증여 방식 선택: "일시 증여" vs "정기금 증여(유기정기금)"
- 유기정기금 계산식 적용 (민법 준용, 할인율 3%)
- "이 방식으로 추가 절세 가능 금액: 약 X만 원" 표시

**주의사항**
- 세무사 검토를 거친 계산 로직 사용 필수
- UI에 "이 결과는 참고용이며 실제 세무 신고는 전문가와 상담하세요" 문구 필수

## 공통 개발 원칙

- 모든 수치는 **출처 명시** + **참고용 면책 문구** 포함
- 금융/세금 관련 계산은 반드시 **"세무사 확인 필요"** 안내
- 물가 데이터 기준값은 **constants 파일로 분리**하여 연 1회 업데이트 용이하게 관리
- v1.2.0 디자인 원칙 준수: 이모지 미사용, Pretendard 폰트, 코랄 컬러 시스템
- 모든 신규 화면은 **모바일 반응형** 필수
